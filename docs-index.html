ManOverMachine API Documentation

Welcome to the ManOverMachine API documentation. Our API provides access to powerful AI and web scraping capabilities through a simple REST interface.

Authentication

All API requests require authentication using an API key. Include your API key in the x-api-key header with each request.

curl -H "x-api-key: your_api_key" ...

User Account Information API

GET /users

The User Account Information API provides access to user account details including status, credits, and profile information.

API Details

  • Base URL: https://xwkwzbjifh.execute-api.us-east-2.amazonaws.com/v1
  • Authentication: API Key in header

Functionality

This API provides the following core functionality:

  • User account verification
  • Credit balance checking
  • Account status verification
  • User profile information retrieval

Authentication

Include your API key in the x-api-key header with each request.

Example Request

curl -X GET https://xwkwzbjifh.execute-api.us-east-2.amazonaws.com/v1/users \
    -H "Content-Type: application/json" \
    -H "x-api-key: your_api_key"

Example Response

{
    "user": {
        "user_id": "59a25033-3915-4ae2-8b1a-29dced12114b",
        "email": "user@example.com",
        "status": "active",
        "credits": 227,
        "plan_type": "credit_based",
        "created_at": 1740246661,
        "updated_at": 1742351793
    }
}

Response Fields

Field Type Description
user_id string Unique identifier for the user
email string User's email address
status string Account status ("active" or "inactive")
credits integer Current credit balance
plan_type string Subscription plan type ("credit_based" or "plan_based")
created_at integer Account creation timestamp (Unix timestamp in US Central time)
updated_at integer Last update timestamp (Unix timestamp in US Central time)

Error Responses

  • 401 - Invalid or missing API key
  • 404 - User not found
  • 500 - Internal server error

ChatGPT API

POST /chatgpt

Send prompts to ChatGPT and receive AI-generated responses.

API Details

  • Base URL: https://1xt5le1gn6.execute-api.us-east-2.amazonaws.com/v1
  • Authentication: API Key in header

Request Parameters

Parameter Type Required Description
prompt string Yes The prompt to send to ChatGPT
model string No Model to use (default: gpt-3.5-turbo)

        "model": "gpt-3.5-turbo",
        "max_tokens": 100

{
    "response": "Artificial intelligence (AI) is a branch of computer science...",
    "model": "gpt-3.5-turbo",
    "max_tokens": 100,
    "credits_remaining": 226
}

NanoBanana API

POST /nanobanana

Generate high-quality AI images using Google's Gemini 2.5 Flash Image Preview model (Nano Banana). Supports both text-to-image generation and advanced image editing with reference images.

Base URL

https://inbhkc0omk.execute-api.us-east-2.amazonaws.com/v1

Operation Modes

  • Text-to-Image Generation: Create images from text descriptions
  • Image Editing: Modify existing images using text prompts and reference images

Parameters

Parameter Type Required Description
prompt string Yes Text description for generation or editing instructions
aspect_ratio string No Image aspect ratio for text-to-image (default: "1:1"). Examples: "16:9", "4:3", "3:2"
reference_image string No Base64 encoded image data for image editing mode
image_mime_type string No MIME type of reference image (default: "image/png")
model string No AI model to use (default: "gemini-2.5-flash-image-preview")

Text-to-Image Example

curl -X POST https://inbhkc0omk.execute-api.us-east-2.amazonaws.com/v1/nanobanana \
    -H "Content-Type: application/json" \
    -H "x-api-key: your_api_key" \
    -d '{
        "prompt": "A bright yellow rubber duck wearing a tiny red pirate hat, floating in a bathtub filled with rainbow-colored soap bubbles",
        "aspect_ratio": "1:1"
    }'

Image Editing Example

curl -X POST https://inbhkc0omk.execute-api.us-east-2.amazonaws.com/v1/nanobanana \
    -H "Content-Type: application/json" \
    -H "x-api-key: your_api_key" \
    -d '{
        "prompt": "Change only the duck to have black and white cow spots. Keep everything else the same.",
        "reference_image": "iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAYAAAAfFcSJAAAADUlEQVR42mP8/5+hHgAHggJ/PchI7wAAAABJRU5ErkJggg==",
        "image_mime_type": "image/png"
    }'

Example Response

{
    "image_data": "iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAYAAAAfFcSJAAAADUlEQVR42mP8/5+hHgAHggJ/PchI7wAAAABJRU5ErkJggg==",
    "image_format": "png",
    "model": "gemini-2.5-flash-image-preview",
    "operation_type": "image editing",
    "aspect_ratio": null,
    "prompt": "Change only the duck to have black and white cow spots. Keep everything else the same.",
    "response_text": "Here's your pirate duck with a Holstein cow pattern!",
    "has_reference_image": true,
    "credits_remaining": 47055.0
}

Response Fields

Field Type Description
image_data string Base64 encoded PNG image data
image_format string Image format (always "png")
model string AI model used for generation
operation_type string "text-to-image generation" or "image editing"
aspect_ratio string Aspect ratio (null for image editing)
prompt string Original prompt used
response_text string Descriptive text about the generated/edited image
has_reference_image boolean Whether a reference image was provided
credits_remaining number Remaining credits in your account

Image Editing Use Cases

  • Style Transfer: "Transform this photo into a watercolor painting"
  • Object Addition: "Add sunglasses to this person"
  • Color Changes: "Change this car from blue to red"
  • Background Replacement: "Put this person in a forest setting"
  • Product Mockups: "Place this logo on a t-shirt"
  • Enhancement: "Improve the lighting in this photo"

Usage Notes

  • Cost: 1 credit per image generation or editing
  • Image Size: Generated images are typically 1-2MB in size
  • Processing Time: 10-15 seconds for generation, 15-20 seconds for editing
  • Format: Images are returned as base64 encoded PNG data
  • Quality: High-resolution images (1024x1024) suitable for professional use
  • Reference Images: Supports PNG, JPEG, and other common formats

JavaScript Examples

Text-to-Image Generation

async function generateImage(prompt, aspectRatio = '1:1') {
    const response = await fetch('https://inbhkc0omk.execute-api.us-east-2.amazonaws.com/v1/nanobanana', {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json',
            'x-api-key': 'your_api_key'
        },
        body: JSON.stringify({
            prompt: prompt,
            aspect_ratio: aspectRatio
        })
    });
    
    const data = await response.json();
    
    // Create image element
    const img = document.createElement('img');
    img.src = 'data:image/png;base64,' + data.image_data;
    document.body.appendChild(img);
    
    return data;
}

Domain Management

The ManOverMachine API includes domain usage tracking and restriction features to prevent unauthorized use of your API key across multiple domains.

Multiple Domain Whitelist

By default, each API key is restricted to usage from a single domain. However, certain users can be whitelisted to use their API key across multiple domains.

Whitelist Feature

  • Users with the allow_multiple_domains flag set to true can use their API key from any domain
  • This feature can only be enabled by administrators through the admin interface
  • Domain usage is still tracked for whitelisted users, but no restrictions are applied

Domain Detection

The API detects source domains through multiple methods (in order of priority):

  1. Explicit x-source-domain header
  2. Origin header
  3. Referer header
  4. API Gateway domain name
  5. Source IP address (as fallback)

Timezone

All timestamps in the ManOverMachine API system are standardized to US Central time (America/Chicago).

Affected Components

  • User details (created_at, updated_at)
  • Credit history timestamps
  • Admin interface displays

Timestamp Format

Timestamps are stored as Unix timestamps (seconds since epoch) but are displayed in US Central time in the admin interface.

Error Handling

The API uses conventional HTTP response codes to indicate the success or failure of an API request.

  • 200 - Success
  • 400 - Bad Request (invalid parameters)
  • 401 - Unauthorized (invalid API key)
  • 402 - Payment Required (insufficient credits)
  • 500 - Internal Server Error

Error Response Format

{
    "statusCode": 400,
    "headers": {
        "Content-Type": "application/json",
        "Access-Control-Allow-Origin": "*",
        "Access-Control-Allow-Headers": "Content-Type,X-Api-Key",
        "Access-Control-Allow-Methods": "OPTIONS,POST"
    },
    "body": {
        "error": "Error message describing what went wrong"
    }
}

POST /webhook

Webhook endpoint for Stripe events (subscription lifecycle, payments).

Handled Events

  • customer.subscription.created - When a new subscription is created
  • customer.subscription.updated - When a subscription is updated
  • customer.subscription.deleted - When a subscription is canceled
  • invoice.payment_succeeded - When a payment is successful (adds credits)
